Architecture Explorer

An 8-tier self-assembling stack — click any tier to explore

TIER 8
Synthesis — The DJ Booth
▶

The FluxSynthesizer wires ALL subsystems together: modules, profiler, selector, tiles, evolution, hot-reload, and system reports. It's the top-level orchestrator that makes everything dance.

Module hierarchy Runtime profiling Adaptive selection Tile composition Self-evolution Hot-swap API System reports
TIER 7
Modules — 8-Level Fractal Hierarchy
▶

Nested containers from TRAIN down to CARD. Atomic hot-reload at any granularity with namespace isolation and SHA-256 checksum trees.

TRAIN CARRIAGE LUGGAGE BAG POCKET WALLET SLOT CARD Fractal reload Checksum trees
TIER 6
Adaptive & Evolution — Self-Improvement
▶

Adaptive (6A): Profiler with 5-level heat classification (FROZEN→HEAT), language selector (Python/TS/Rust/C/SIMD), compiler bridge. Evolution (6B): Genome snapshots, pattern mining, 7 mutation strategies, correctness validation, multi-generation loop.

Heat map Language rec Genome DNA Pattern mining 7 mutators Validation Fitness: speed + modularity + correctness
TIER 5
Tiles — 35 Composable Patterns
▶

Reusable computation tiles across 6 categories. DAG composition via TileGraph, FIR emission via blueprints, registry with fuzzy search and topological ordering.

COMPUTE(8) MEMORY(6) CONTROL(6) A2A(6) EFFECT(3) TRANSFORM(6) DAG graph Kahn's sort
TIER 4
Agent Runtime — Trust & Scheduling
▶

Agent wrapper around the VM interpreter. AgentRuntime orchestrator with multi-agent support, trust-gated messaging, and CLI (compile/run/test).

Agent wrapper Orchestrator Trust gates CLI
TIER 3
A2A Protocol — Agent Communication
▶

4 typed message kinds, 3 channel patterns, capability-based registry, 4-step trust handshaking, binary serialization with batch support. 32 native A2A opcodes in bytecode.

TELL ASK DELEGATE BROADCAST Trust handshake Binary codec
TIER 2
Supporting — Optimizer, JIT, Types, Stdlib, Security
▶

Optimizer: constant folding, DCE, inlining. JIT: LRU cache, execution tracing, register allocation. Types: C/Python/Rust unification, generics. Stdlib: 26 intrinsics. Security: capability tokens, sandbox.

Optimizer JIT Type system Stdlib (26 fns) Security Hot reload Cost model
TIER 1
Core Pipeline — FLUX.MD → Bytecode → VM
▶

L0 Parser: Structured markdown with frontmatter, code blocks, agent directives. L1 Frontends: C + Python → FIR. L2 FIR: SSA IR with 15 types, 42 instructions. L3 Bytecode: 104 opcodes, 6 encoding formats. L4 VM: 64-register fetch-decode-execute.

FLUX.MD parser C frontend Python frontend FIR (SSA) 15 types 42 instructions 104 opcodes 64 registers Linear memory
